改造概览

原项目 lfenghx/skill_agent 为 Dify Tool 插件，内置 2 个工具。本次改造聚焦于系统提示词重构（提取为独立模块，用英文重写）、Dify 插件声明修正、代码去重重构与补齐打包能力。

插件架构

Skill_Agent 是单 Dify 插件，内含两个 Tool：skill_agent（核心 Agent，渐进式披露调度）与 tm（技能管理，Skill 包 CRUD）。本次改造保持架构不变，聚焦代码质量提升。

图 1 — 改造后双层架构：Agent + Tool 两大插件通过共享 skills_root 协作

skill_agent Tool（核心 Agent）

  • 10 个内部函数通过 TOOL_SCHEMAS 注册为 LLM tools
  • 渐进式披露流程：metadata → list files → read file → run command
  • 支持 function call（流式）与 JSON 协议两种 LLM 交互模式
  • session.storage 持久化：跨轮次 resume、history_turns 记忆窗口
  • temp 会话目录：write/read/run_temp + export_temp_file 文件交付

TM Tool（技能管理器）

  • command 参数驱动：查看技能 / 新增技能 / 删除技能N / 下载技能N
  • zip 上传解压：_safe_extract_zip 逐文件 is_within_dir 防 Zip Slip
  • SKILL.md 自动发现：解压后 _find_skill_folders 定位技能目录
  • 下载导出：shutil.make_archive + create_blob_message 文件回传
  • 文件处理：blob 优先（SDK 转换），fallback URL+下载

Manifest 与 YAML 修正 关键修复

原项目的 skill_agent.yaml 声明了 files 参数（type: files），导致 Dify 插件校验报错。同时 endpoint 权限误开、参数重复定义。

tools/skill_agent.yaml — 参数修复 修复 
-   - name: files
-     type: files
-     required: false      # ← Dify 校验报错，移除
-     label:
-       en_US: Files
-       zh_Hans: 上传文件
-     llm_description: ...
-     form: llm            # ← 类型错误，应为 form

+   - name: system_prompt
+     type: string
+     required: false
+     default: "你是一个基于 Skill
+       渐进式披露模式执行任务的智能代理"
+     form: form

核心修复：移除不合规的 files 参数；补全 system_prompt 参数声明

manifest.yaml — 权限与版本修正 修正 
# resource.permission.endpoint
-   endpoint:
-     enabled: true        # ← 原项目误开，无使用场景
+   endpoint:
+     enabled: false

# meta.version
- meta:
-   version: 0.0.4        # ← 与 manifest 版本冲突
+ meta:
+   version: 0.0.1

权限收窄：关闭不使用的 endpoint 权限；版本修正：meta.version 与 plugin version 解耦

工具层改动 重构 + 增强

skill_agent 和 TM 两个工具的核心逻辑改动：消除 tool_calls 处理重复、英文化提示词、TM 文件 URL 智能解析。

TM.py 增强 — 文件处理链路 +49 LOC

原 TM.py 只处理绝对 HTTP URL，在 Dify 容器内部环境下文件相对 URL 会失败。

改动点改造前改造后
_resolve_file_url() 不存在 新增函数：读取 DIFY_INNER_API_URL 环境变量，将 /files/xxx 拼接为完整 URL
get_file_content() 直接用原始 URL 发起 Request 先 _resolve_file_url 解析，异常信息包含完整 URL
文件 blob 优先级 只从 URL 下载 优先尝试 file_item.blob（SDK 转换后的 File 对象），fallback URL 下载
错误提示 "未检测到上传的 zip 文件，请提供 files 参数" 详细 3 条排查指引（文件上传节点、参数绑定、Chat 上传按钮）

skill_agent.py 重构 — 去重 + 英文化 -41 LOC

原 skill_agent.py 中 function call 与 JSON 协议两套 tool_calls 处理逻辑高度重复（~80 行几乎相同的代码块），且 prompt 字符串硬编码在 Python 中。

改动点改造前改造后
tool_calls 重复逻辑 function call 和 JSON 协议各写一套工具调度代码（~80 行 × 2） 提取公共执行逻辑到运行时模块，只保留协议差异部分
prompt 硬编码 System Prompt、错误提示、状态消息、续跑文本全部内联写在代码中 提取到 utils/skill_agent_prompts.py 独立模块，英文重写，分 6 类 27 个常量
system_prompt 参数 硬编码："你是一个基于 Skill 渐进式披露模式..." 补全 system_prompt 的 YAML 声明
compact 函数 简单截断，无上下文保护 保留 system message + 尾部 memory_turns × 4 轮，防止丢失核心指令

打包脚本 全新

原项目无打包工具。新增 scripts/package_plugin.py（+121 LOC），纯 Python 实现，无需 Dify CLI。 

# 用法
python scripts/package_plugin.py . -o skill_agent.difypkg

# 功能
- 读取 .difyignore 排除文件
- 自动忽略 .git/ 目录
- 校验 manifest.yaml 存在
- 输出 .difypkg（即标准 zip）
